# 🛒 Superstore Sales Analysis — SQL Project

## 📌 Project Overview
A comprehensive SQL-based analysis of the Superstore 
retail dataset to uncover revenue trends, customer 
behavior patterns, and product performance insights 
that can drive business decisions.

---

## 🎯 Business Problem
Retail businesses often struggle to answer critical 
questions like:
- Which products and categories drive the most revenue?
- Which customers are most valuable?
- Which regions are underperforming?
- How is revenue trending month over month?

This project answers all of these using only SQL.

---

## 📂 Dataset
- **Source:** [Kaggle — Superstore Dataset](https://www.kaggle.com/datasets/vivek468/superstore-dataset-final)
- **Size:** ~10,000 rows | 21 columns
- **Period:** 2014 – 2017
- **Domain:** Retail / E-commerce

**Key Columns:**
| Column | Description |
|---|---|
| Order ID | Unique order identifier |
| Customer ID | Unique customer identifier |
| Category | Product category |
| Sub-Category | Product sub-category |
| Sales | Revenue from order |
| Profit | Profit from order |
| Region | Geographic region |
| Order Date | Date of purchase |

---

## 🛠️ Tools Used
- **Database:** PostgreSQL 
- **IDE:** pgAdmin 4
- **Version Control:** Git + GitHub
